## Getting started

This project was generated by hand (not via `ng new`) in an offline sandbox,
so dependencies are not yet installed. On your machine, with internet access:

```bash
npm install
npm start
```

Then open http://localhost:4200.

## What's implemented (Phases 1â€“5)

- Angular 20 standalone app, strict TypeScript, Signals-based state
- Pure TypeScript Game Engine (`src/app/engine`) with a Command pattern
  (`SelectRegionCommand`, `CollectIncomeCommand`, `PurchaseUnitCommand`,
  `AdvancePhaseCommand`, `EndTurnCommand`, `DeployUnitCommand`,
  `MoveUnitCommand`, `LoadUnitCommand`, `UnloadUnitCommand`) and an event
  system, completely independent of Angular
- Signal Store (`src/app/state`) that is the *only* bridge between UI and Engine
- JSON data loading (`src/app/data/countries.json`, `factions.json`,
  `economy.json`, `units.json`, `straits.json`, `sea-zones.json`,
  `starting-deployment.json`) â€” no hardcoded gameplay values
- HTML5 Canvas world map renderer with click/hover region selection,
  neighbor highlighting, per-region unit silhouettes (one icon per unit
  type, faction-colored, with a count badge for stacks), and digitized
  sea zone markers
- Region info side panel, now including the units stationed in the
  selected region
- Full turn cycle per PROJECT_RULES.md v2.0: Buy Units â†’ Cyber Attack â†’
  Attack Moves â†’ Attack â†’ Tactical Moves â†’ Place New Units â†’ Collect
  Income (last, funds next turn), rotating through non-eliminated players
  and incrementing the turn number once per full round. Collect Income is
  capped at once per turn.
- 12-unit catalog (Infantry, Tank, Helicopter, Submarine, Stealth Boat,
  Destroyer, Fighter, Rocket System, Land Transport, Aircraft Carrier,
  Missile A/B) with a data-driven per-player Reserve
- Each faction starts the game with units already deployed across its own
  regions/sea zones (`data/starting-deployment.json`), matching the
  reference manual's per-faction totals table
- One-hop movement (Attack Moves + Tactical Moves phases) with per-turn
  movement points, enemy-occupied regions blocking movement, strait
  chokepoints (land bridges only when both sides are owned by the mover),
  and a basic transport/load/unload system where embarked units travel
  with their transport and can't move independently

## What's not implemented yet

The Cyber Attack Phase, Citizen/Public Spending, and the detailed per-unit
combat rules (Tank capture-through, Submarine submerge, Fighter landing
restrictions, etc. â€” PROJECT_RULES.md sections 5-6, 20-30) are documented
but NOT yet coded â€” they require the Combat engine (Phase 6) and Cyber
Warfare system (Phase 9), neither of which exist yet. Attack/Combat are
currently pass-through phases: units can move into any non-enemy-owned
region, but nothing happens when they'd otherwise contest one. Naval
movement currently shares the land connectivity graph rather than a
dedicated sea-lane system; sea zones exist as data (digitized from the
map image, best-effort) but aren't yet wired into movement routing. Per
the project's golden rule, these are intentionally not started until
Phase 5 is confirmed working.
